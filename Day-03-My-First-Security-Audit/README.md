# Day 03 - My First Security Audit: Auditing Myself

## Objective
Understand what a security audit is and learn a simple methodology to assess the security of any system, including my own digital presence.

---

## Lesson of the Day

Today, I learned that a security audit is like a health check-up.

A security audit helps answer questions such as:

- How secure is it right now?
- How secure should it be?
- What risks or weaknesses exist?
- What improvements are needed?

I also created my own personal security audit report and saved it in my learning folder.

---

## Five-Step Audit Methodology

### 1. Scope
Define what is being audited.

Examples:
- Email accounts
- Social media accounts
- Devices
- Passwords
- Online presence

### 2. Intelligence
Gather information about the target.

Examples:
- Number of accounts
- Security settings enabled
- Password practices
- Recovery methods

### 3. Exposure
Identify what is publicly exposed or vulnerable.

Examples:
- Breached accounts
- Reused passwords
- Missing MFA
- Publicly available information

### 4. Severity
Determine the impact and risk level of each issue.

Examples:
- Low Risk
- Medium Risk
- High Risk
- Critical Risk

### 5. Remediation
Create a plan to fix the identified issues.

Examples:
- Change passwords
- Enable Multi-Factor Authentication (MFA)
- Remove unnecessary public information
- Improve account recovery methods

---

## What I Learned

- A security audit is a structured way to evaluate security.
- Audits help identify gaps between current security and desired security.
- The five-step methodology can be used to audit almost anything.
- Understanding exposure and severity helps prioritize security improvements.
- Remediation is the most important step because it turns findings into action.
