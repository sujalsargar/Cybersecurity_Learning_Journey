## The History of Password Rules

In **2003**, password guidelines encouraged users to:
- Use uppercase letters
- Use lowercase letters
- Include numbers
- Include special characters
- Change passwords every 90 days

These recommendations became common across many organizations.

Later, it was discovered that these rules were based on guidance that wasn't supported as strongly as many believed. People often created predictable passwords or made only small changes every 90 days, which did not significantly improve security.

The guidance was written by **Bill Burr**, who later acknowledged that some of the recommendations were not as effective as intended.

---

## Updated NIST Recommendations (2017)

In **2017**, the **National Institute of Standards and Technology (NIST)** updated its password guidance.

The recommendations include:

- Prefer **long passwords or passphrases**.
- Do **not** force users to change passwords periodically unless there is evidence of compromise.
- Block commonly used and weak passwords.
- Encourage unique passwords for every account.

These recommendations focus on improving real-world security while making passwords easier for users to manage.

## Common Password Attacks

### 1. Credential Stuffing
Attackers use usernames and passwords leaked from previous data breaches to try logging into other websites, relying on people reusing passwords.

---

### 2. Phishing
Attackers trick users into revealing their passwords through fake websites, emails, or messages that appear legitimate.

---

### 3. Dictionary Attack
Attackers automatically try passwords from a list of common words, phrases, and leaked passwords.

---

### 4. Password Spraying
Instead of trying many passwords on one account, attackers try a few common passwords (such as `Password123`) across many different accounts to avoid account lockouts.

---

## Three Password Habits I Learned

### 1. Use a Different Password for Every Account
If one account is compromised, the others remain protected.

### 2. Use a Password Manager
A password manager helps generate, store, and manage strong, unique passwords securely.

### 3. Enable Two-Factor Authentication (2FA)
Adding a second verification step greatly improves account security, especially for important accounts like email, banking, and social media.

---

## What I Learned

- Password length is often more important than unnecessary complexity.
- Long passphrases are easier to remember and harder to crack.
- Modern password guidance focuses on usability as well as security.
- Password reuse creates major security risks.
- Password managers and 2FA are essential security practices.

---

## Key Takeaways

✅ Use long and unique passwords.

✅ Avoid reusing passwords across multiple accounts.

✅ Use a trusted password manager.

✅ Enable Two-Factor Authentication (2FA) whenever possible.

✅ Understand common password attacks such as Credential Stuffing, Phishing, Dictionary Attacks, and Password Spraying.
